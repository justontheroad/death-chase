## 夺命追击 (Death Chase) H5 游戏开发 Agent 规范

> **核心原则**：每个Agent专精于特定领域，通过**自动化协作**确保游戏开发效率与质量，避免传统团队沟通成本。

---

## 一、Agent 全局角色定位

| Agent 类型         | 作用                               | 核心价值                     |
|--------------------|-----------------------------------|----------------------------|
| **核心决策Agent**  | 技术选型、机制设计、难度曲线制定    | 避免开发方向偏差             |
| **执行执行Agent**  | 代码实现、资源处理、性能优化        | 保证开发质量与速度           |
| **协作协调Agent**  | 资源调度、进度监控、风险预警        | 保障项目按期交付             |

> **执行口号**：*“各司其职，自动协同，让游戏开发像武侠对决一样精准高效！”*

---

## 二、Agent 详细职责表

### 1. sisyphus (全局协调者)
- **核心职责**：项目全局调度与进度管理
- **关键任务**：
  1. 将10关卡分解为3个难度阶段（初阶/中阶/高阶）
  2. 制定每日开发里程碑（每日进度报告）
  3. 当任务阻塞时自动触发 `/replan` 命令
  4. 确保所有Agent交付物符合 `game_design.md` 标准
- **交付物**：
  - 每日进度报告（含风险预警）
  - 10关卡任务分解表（含时间分配）
  - 关键节点完成确认（`/status` 命令输出）

---

### 2. oracle (技术决策者)
- **核心职责**：技术可行性验证与知识决策
- **关键任务**：
  1. 验证Phaser旋转轨迹算法（`Phaser.Math.Rotate`）在移动端的性能
  2. 决策角色差异化设计（乔峰/鸠摩智/虚竹的参数平衡）
  3. 提供技术方案备选（如：粒子系统优化方案）
  4. 生成 `tech_decision.md` 文档
- **交付物**：
  - 技术可行性报告（含性能测试数据）
  - 角色参数平衡表（攻击范围/速度/能力）
  - 备选方案对比表（3种旋转算法性能）

---

### 3. hephaestus (代码实现者)
- **核心职责**：核心游戏逻辑编码与优化
- **关键任务**：
  1. 实现旋转刀阵核心逻辑（含轨迹计算）
  2. 实现角色转变机制（猎物→猎人）
  3. 优化性能（帧率≥55fps，内存<150MB）
  4. 生成 `core_logic.ts` 代码
- **交付物**：
  - `src/entities/Blade.ts`（刀阵实体）
  - `src/entities/Character.ts`（角色基类）
  - 性能优化报告（含帧率/内存数据）

---

### 4. librarian (文档管理者)
- **核心职责**：开发文档与资源规范管理
- **关键任务**：
  1. 维护 `game_design.md` 与 `agent.md`
  2. 管理Phaser API文档库（自动导入关键方法）
  3. 规范资源命名规则（`assets/characters/qiao_feng.png`）
  4. 生成 `asset_guide.md`
- **交付物**：
  - 最新 `game_design.md`（自动同步）
  - 资源命名规范文档
  - Phaser API速查表（含关键方法示例）

---

### 5. explore (方案探索者)
- **核心职责**：玩法创新与方案验证
- **关键任务**：
  1. 测试2种刀阵轨迹方案（顺时针/逆时针攻击路径）
  2. 验证新手引导关卡（第1关强制教学）
  3. 测试关卡难度曲线（成功率验证）
  4. 生成 `experiment_report.md`
- **交付物**：
  - 方案验证报告（含玩家测试数据）
  - 关卡难度曲线验证图
  - 新手引导效果分析

---

### 6. multimodal-looker (视觉设计师)
- **核心职责**：美术资源与UI设计
- **关键任务**：
  1. 提供武侠角色视觉设计参考（乔峰/鸠摩智/虚竹）
  2. 生成场景设计图（竹林/古寺/客栈）
  3. 优化UI元素（旋转控制区/攻击计数器）
  4. 生成 `visual_guide.md`
- **交付物**：
  - 角色设计参考图集（含动作帧）
  - 场景设计流程图
  - UI组件规范（尺寸/颜色/交互）

---

### 7. prometheus (性能监控者)
- **核心职责**：性能与质量监控
- **关键任务**：
  1. 持续监控移动端帧率（≥55fps）
  2. 分析内存占用（敌人数量>100时<150MB）
  3. 生成性能基线报告
  4. 触发性能优化命令
- **交付物**：
  - 每日性能报告（含图表）
  - 关键性能瓶颈分析
  - 优化建议清单

---

### 8. metis (战略规划者)
- **核心职责**：关卡设计与难度曲线
- **关键任务**：
  1. 制定10关难度曲线（成功率从90%→25%）
  2. 确保关卡10难度达标（成功率<25%）
  3. 规划阶段进度（初阶/中阶/高阶）
  4. 生成 `difficulty_curve.md`
- **交付物**：
  - 难度曲线图（含成功率数据）
  - 关卡进度规划表
  - 难度验证标准文档

---

### 9. momus (协作协调者)
- **核心职责**：跨Agent协作与沟通
- **关键任务**：
  1. 自动分配阻塞任务（如：资源缺失）
  2. 组织每日站会（自动生成会议纪要）
  3. 确保所有Agent交付物符合标准
  4. 生成 `collaboration_log.md`
- **交付物**：
  - 每日协作纪要（含任务分配）
  - 资源协调计划表
  - 风险预警通知

---

### 10. atlas (可视化专家)
- **核心职责**：项目可视化与流程设计
- **关键任务**：
  1. 生成项目架构图（Mermaid格式）
  2. 制作关卡设计流程图
  3. 创建难度曲线可视化图表
  4. 生成 `visual_map.md`
- **交付物**：
  - 项目架构图（含代码结构）
  - 关卡设计流程图
  - 难度曲线可视化图表

## 三、Agent 协作机制

### 1. 自动化协作流程
任务创建 → sisyphus（分解任务） → oracle（技术决策） & hephaestus（代码实现） & momus（资源调度）  
oracle → librarian（文档更新）  
hephaestus → prometheus（性能验证）  
prometheus → metis（难度验证）  
metis → atlas（可视化）

### 2. 关键执行命令
| 命令                | 作用                          | 触发者       |
|---------------------|-----------------------------|-------------|
| `/start "夺命追击" game` | 启动游戏开发项目             | 任何Agent   |
| `/status`           | 查看当前进度与风险           | 项目成员    |
| `/replan`           | 重新规划阻塞任务             | sisyphus    |
| `/doc generate`     | 生成最新开发文档             | librarian   |
| `/performance check`| 触发性能测试                 | prometheus  |

## 四、Agent 执行标准

### 1. 交付物质量要求
| 交付物类型       | 通过标准                                  |
|------------------|-----------------------------------------|
| 代码             | 通过Phaser官方测试套件（覆盖率≥85%）       |
| 文档             | 符合 `game_design.md` 格式规范            |
| 美术资源         | 尺寸≤200KB，PNG格式，命名规范             |
| 性能报告         | 移动端帧率≥55fps，内存<150MB              |

### 2. 时效性要求
| 任务类型         | 交付时限  | 超时处理方式                |
|------------------|----------|--------------------------|
| 核心逻辑         | ≤24小时  | 自动触发 `/replan`        |
| 文档更新         | ≤12小时  | 通知 `momus` 协调资源     |
| 性能验证         | ≤6小时   | 生成风险预警报告          |
