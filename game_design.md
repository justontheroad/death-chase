# 夺命追击 (Death Chase) - 游戏设计文档

## 1. 游戏概述

**夺命追击 (Death Chase)** 是一款基于H5浏览器的大型多人在线动作生存游戏。玩家控制武侠角色在开放世界中生存，通过收集武器、消灭敌人来不断变强。

### 核心玩法
- **旋转武器系统**: 武器围绕角色旋转，数量代表战斗力
- **生存进化**: 收集道具增加武器数量，变得更强
- **战斗力对决**: 武器多的消灭武器少的
- **开放世界**: 8000x6000像素大地图自由探索

## 2. 游戏特性

### 2.1 核心特性

| 特性 | 说明 |
|------|------|
| ⚔️ **旋转武器** | 武器围绕角色做圆周运动，形成防护圈 |
| 📈 **成长系统** | 收集道具增加武器数量，提升战斗力 |
| ⚡ **实时战斗** | 武器碰撞即战斗，强者生存 |
| 👹 **智能敌人** | 敌人自动追踪，各自拥有随机武器数量 |
| 🗺️ **开放世界** | 8000x6000像素超大可探索地图 |
| 📷 **相机跟随** | 平滑跟随玩家，0.7倍缩放视野 |
| 💎 **奖励系统** | 消灭敌人掉落绿色奖励，+25经验 |
| 🏆 **等级系统** | UI显示当前武器数量和等级 |
| 💀 **游戏结束** | 武器少于敌人时被消灭 |

### 2.2 战斗机制

#### 战斗力规则
```
玩家武器数量 > 敌人武器数量 = 玩家胜利，敌人被消灭
玩家武器数量 < 敌人武器数量 = 敌人胜利，游戏结束
玩家武器数量 = 敌人武器数量 = 平局，游戏结束（保守策略）
```

#### 武器系统
- **初始武器**: 3把剑（等级1）
- **武器类型**: 剑、斧、矛、锤、匕首
- **武器等级**: 1-5级，等级越高属性越强
- **武器范围**: 半径120像素（根据武器类型和等级调整）
- **旋转速度**: 每秒3圈（根据武器类型和等级调整）
- **增长方式**: 收集武器道具或绿色奖励

##### 武器类型属性
| 武器类型 | 伤害 | 范围 | 速度 | 特点 |
|---------|------|------|------|------|
| 剑 | 10 | 1.0 | 1.0 | 平衡型 |
| 斧 | 15 | 0.8 | 0.7 | 高伤害，低速度 |
| 矛 | 8 | 1.3 | 0.9 | 长范围，中等速度 |
| 锤 | 20 | 0.6 | 0.5 | 超高伤害，低速度 |
| 匕首 | 5 | 0.9 | 1.3 | 低伤害，高速度 |

##### 武器等级效果
| 等级 | 颜色 | 伤害加成 | 外观变化 |
|------|------|----------|----------|
| 1 | 白色 | +0% | 基础大小 |
| 2 | 金色 | +20% | 稍微变大 |
| 3 | 绿色 | +40% | 明显变大 |
| 4 | 蓝色 | +60% | 更大 |
| 5 | 紫色 | +80% | 最大 |

#### 敌人系统
- **初始数量**: 8个敌人
- **武器随机**: 1-5把随机武器，武器类型和等级随机
- **AI行为**: 自动追踪玩家，保持100-800像素距离
- **颜色标识**: 红色显示武器数量
- **武器系统**: 敌人也会使用不同类型和等级的武器，等级根据敌人强度调整

## 3. 游戏机制

### 3.1 控制方式

| 操作 | 功能 |
|------|------|
| 🖱️ **点击屏幕** | 移动到点击位置 |
| 📱 **触摸拖动** | 平滑移动角色 |

### 3.2 道具系统

#### 武器道具（Weapon Power-Up）
- **外观**: 不同等级的旋转武器，颜色和大小根据等级变化
- **效果**: 立即升级武器，并随机更换武器类型
- **生成**: 地图随机位置，初始5个
- **动画**: 上下浮动
- **等级分布**: 等级1 (50%), 等级2 (30%), 等级3 (15%), 等级4 (4%), 等级5 (1%)

#### 绿色奖励（Reward）
- **外观**: 绿色旋转剑
- **来源**: 消灭敌人后掉落
- **效果**: +25经验，可升级增加武器
- **收集**: 触碰即可收集

### 3.3 成长路线

```
开始: 3把剑（等级1）
↓
收集武器道具 → 升级武器并随机更换武器类型
↓
消灭敌人 → 掉落绿色奖励
↓
收集绿色奖励 → +25经验
↓
经验满 → 自动升级武器数量
↓
继续变强 → 成为最强王者
```

### 3.4 游戏流程

1. **开始游戏**: 初始3把剑（等级1），8个敌人
2. **探索世界**: 移动寻找武器道具和敌人
3. **收集成长**: 拾取武器道具升级武器并更换武器类型
4. **战斗对抗**: 武器多且等级高的消灭武器少且等级低的
5. **获取奖励**: 消灭敌人掉落绿色奖励
6. **持续变强**: 武器数量和等级不断增加
7. **成为王者**: 消灭所有敌人获胜
8. **或被消灭**: 武器少于或等级低于敌人时失败

## 4. 技术规格

### 4.1 平台要求
- **目标平台**: 移动浏览器（H5）
- **性能目标**: ≥55fps帧率，<150MB内存使用
- **渲染器**: Canvas（避免WebGL残影问题）
- **分辨率**: 800x600基础，支持缩放

### 4.2 技术栈
- **引擎**: Phaser.js 3.55.2
- **语言**: TypeScript
- **构建**: Webpack 5
- **资源**: SVG矢量图形

### 4.3 性能优化
- ✅ `clearBeforeRender: true` - 清除残影
- ✅ `transparent: false` - 不透明背景
- ✅ 平滑相机跟随 (0.08插值)
- ✅ 0.7倍缩放减少绘制区域
- ✅ 16ms间隔碰撞检测

## 5. 游戏世界

### 5.1 世界设定
- **尺寸**: 8000 x 6000 像素
- **背景**: 竹林主题
- **网格**: 400px间隔辅助线
- **边界**: 物理边界限制

### 5.2 实体系统

#### 玩家 (Player)
- **角色**: 乔峰/鸠摩智/虚竹
- **初始武器**: 3把
- **移动速度**: 300像素/秒
- **外观**: 蓝色标识

#### 敌人 (Enemy)
- **数量**: 动态生成，最少5个
- **武器**: 随机1-5把
- **速度**: 50-80像素/秒
- **AI**: 追踪玩家，保持一定距离
- **外观**: 红色标识，显示武器数量

#### 武器 (Blade)
- **旋转**: 围绕角色圆周运动，武器尖端指向旋转方向
- **半径**: 120像素（根据武器类型和等级调整）
- **速度**: 2-4圈/秒（根据武器类型和等级调整）
- **方向**: 顺时针/逆时针随机
- **类型**: 剑、斧、矛、锤、匕首
- **等级**: 1-5级，等级越高属性越强

## 7. 游戏平衡

### 7.1 难度曲线
- **前期**: 3把武器 vs 1-2把敌人（优势）
- **中期**: 需要收集道具增长
- **后期**: 面对5把武器的强敌

### 7.2 策略要素
1. **避强击弱**: 先消灭武器少的敌人
2. **快速成长**: 优先收集金色道具
3. **风险控制**: 避免与武器多的敌人碰撞
4. **资源管理**: 收集绿色奖励持续成长
